博客搭建，Windows

使用hugo+GitHub搭建个人博客（Windows10环境下）

一直以来我都希望能有一个属于自己的博客，在上面分享自己的东西，但是限制于自己的技术问题，我一直没有成功过，所以也挺遗憾的。直到我在B站上面看到了手把手教你从0开始搭建自己的个人博客 |第二种姿势 | hugo这个视频之后，我发觉我可能要实现搭建个人博客这个愿望了
但是由于羊哥使用的是macOS，我手头只有Windows10，所以还是比较难以摸索的，于是我就在评论区找别人的博客，终于功夫不负有心人，让我找到了一篇Hugo+github搭建个人博客（windows10）,我跟着他的步骤一步一步搞了下去，还是可以完美的搭建出一个个人博客的，但是其中仍然存在一些问题，所以我还是决定自己写一篇文章来分享一下我的做法

准备工作

  • 了解GitHub和git(不了解可以跟着教程走)
  • 注册一个GitHub账号
  • 安装Hugo和Git
  • 安装VS code(用于部署过程中编辑各种文本，windows自带的记事本真的不行,有一说一这个记事本真的烦人)

安装Git

Git的安装还是比较简单的，一般情况下，一直点击下一步就好，应该不会遇到什么困难，参见Git安装教程。

安装Hugo

这里就讲一下比较简单的安装方法

1.下载Hugo,去Hugo官方GitHub下载https://github.com/gohugoio/hugo/releases，在这里之间以使用0.57.2以及以下版本 2.将下载的安装包解压到文件夹之中，最好是经常不动的文件夹，而且一定要记住==文件夹位置==
3.将Hugo添加到系统环境变量
       1.首先找到系统的环境变量

       2.添加系统环境变量，依此：点击环境变量，找到系统变量中的path，点击编辑，然后点击新建，在使用浏览按钮选中文件夹，即可使用hugo。（选中到hugo.exe所在的文件夹即可，不需要选中hugo.exe）

4.接下来来到cmd测试Hugo是否安装完成

hugo version

如果得到以下响应则证明安装完成

Hugo Static Site Generator v0.73.0-428907CC windows/amd64 BuildDate: 2020-06-23T16:32:10Z

创建并配置站点

==以下使用的命令均在以管理员方式运行的cmd命令行窗口下执行 以下所有的“XXXX”均为占位符，你可以替换为属于你自己的文件名==

  • 创建站点命令（使用cmd命令手册中的CD命令，选择一个文件夹，在其中存放站点的相关文件，==文件路径不要有中文==）
hugo new site xxx

       这条命令执行完毕后，会生成一个名字为"xxx"的文件夹，我们在这将其称之为根目录。

下载并配置主题（以tranquilpeak主题为例）

下载主题

你可以在Hugo的主题库中选择你喜欢的主题，这里以tranquilpeak主题作为教程示例。
如果下载速度特别慢，可以参考解决github下载/克隆速度太慢问题这篇文章。并且配置完毕后在cmd中输入

ipconfig/flushdns

在根目录下使用CD命令来到themes文件夹下，并且在tranquilpeak主题下很容易找到clone命令如下：

cd themes
git clone https://github.com/kakawait/hugo-tranquilpeak-theme.git tranquilpeak

#以下为注释说明，可以在clone的整句命令后加上你要clone到的文件夹名字，默认是hugo-tranquilpeak-theme
#可是默认的文件夹名称实在是太长了，为了方便使用，可以在命令后边加上空格然后指定为一个短点儿的文件夹名字，比如这里的tranquilpeak

正常情况下应该很快就能clone完毕了，如果不行可以尝试拉取该主题的以前版本，然后在拉取全部内容，当然这只是权宜之计，clone慢还是说明host文件没有修改好，建议再去修改。

git clone https://github.com/kakawait/hugo-tranquilpeak-theme.git tranquilpeak --depth 1
#等待这条命令走完后访问该主题的文件夹
cd tranquilpeak
#然后clone全部内容
git fetch --unshallow

在主题配置中加入自定义的js插件以后，添加的点击爱心特效插件的代码如下：

(function(window,document,undefined){
    var hearts = [];
    
    window.requestAnimationFrame = (function(){
        return window.requestAnimationFrame || 
        window.webkitRequestAnimationFrame ||
        window.mozRequestAnimationFrame ||
         window.oRequestAnimationFrame ||
         window.msRequestAnimationFrame ||
         function (callback){
             setTimeout(callback,1000/60);
         }
    })();
    
    init();
 
    function init(){
        css(".heart{width: 10px;height: 10px;position: fixed;background: #f00;transform: rotate(45deg);-webkit-transform: rotate(45deg);-moz-transform: rotate(45deg);}.heart:after,.heart:before{content: '';width: inherit;height: inherit;background: inherit;border-radius: 50%;-webkit-border-radius: 50%;-moz-border-radius: 50%;position: absolute;}.heart:after{top: -5px;}.heart:before{left: -5px;}");
        attachEvent();
        gameloop();
    }
 
    function gameloop(){
        for(var i=0;i<hearts.length;i++){
            if(hearts[i].alpha <=0){
                document.body.removeChild(hearts[i].el);
                hearts.splice(i,1);
                continue;
             }
 
             hearts[i].y--;
             hearts[i].scale += 0.004;
             hearts[i].alpha -= 0.013;
             hearts[i].el.style.cssText = "left:"+hearts[i].x+"px;top:"+hearts[i].y+"px;opacity:"+hearts[i].alpha+";transform:scale("+hearts[i].scale+","+hearts[i].scale+") rotate(45deg);background:"+hearts[i].color;
        }
 
        requestAnimationFrame(gameloop);
    }
 
    function attachEvent(){
        var old = typeof window.onclick==="function" && window.onclick;
        window.onclick = function(event){
            old && old();
            createHeart(event);
        }
    }
 
    function createHeart(event){
        var d = document.createElement("div");
        d.className = "heart";
        hearts.push({
            el : d,
            x : event.clientX - 5,
            y : event.clientY - 5,
            scale : 1,
            alpha : 1,
            color : randomColor()
        });
 
        document.body.appendChild(d);
    }
 
    function css(css){
        var style = document.createElement("style");
        style.type="text/css";
        try{
            style.appendChild(document.createTextNode(css));
        }
        catch(ex){
            style.styleSheet.cssText = css;
        }
 
        document.getElementsByTagName('head')[0].appendChild(style);
    }
 
    function randomColor(){
        return "rgb("+(~~(Math.random()*255))+","+(~~(Math.random()*255))+","+(~~(Math.random()*255))+")";
    }
    
})(window,document);

       雪花的插件代码如下：

(function($){
    $.fn.snow = function(options){
    var $flake = $('<div id="snowbox" />').css({'position': 'absolute','z-index':'9999', 'top': '-50px'}).html('&#10052;'),
    documentHeight  = $(document).height(),
    documentWidth   = $(document).width(),
    defaults = {
        minSize     : 10,
        maxSize     : 20,
        newOn       : 1000,
        flakeColor  : "#AFDAEF" /* 此处可以定义雪花颜色，若要白色可以改为#FFFFFF */
    },
    options = $.extend({}, defaults, options);
    var interval= setInterval( function(){
    var startPositionLeft = Math.random() * documentWidth - 100,
    startOpacity = 0.5 + Math.random(),
    sizeFlake = options.minSize + Math.random() * options.maxSize,
    endPositionTop = documentHeight - 200,
    endPositionLeft = startPositionLeft - 500 + Math.random() * 500,
    durationFall = documentHeight * 10 + Math.random() * 5000;
    $flake.clone().appendTo('body').css({
        left: startPositionLeft,
        opacity: startOpacity,
        'font-size': sizeFlake,
        color: options.flakeColor
    }).animate({
        top: endPositionTop,
        left: endPositionLeft,
        opacity: 0.2
    },durationFall,'linear',function(){
        $(this).remove()
    });
    }, options.newOn);
    };
})(jQuery);

thumbnailImagePosition: top

autoThumbnailImage: yes

# 封面图片，如果选用本地文件的话，和缩略图同样的路径，21：9的图片效果很好看

coverCaption: "文章cover图片描述"

coverSize: full

hugo new post/xxx.md

待本地启动后，可以继续修改这篇文档的内容，在需要的文件夹中加入需要的图片，然后调整相关的参数，看看图片的几个功能是否正常或者符合预期。

启动本地预览

建议在配置文件中设置好主题，或者使用 –t指令指定主题，在站点的根目录下使用命令进行本地启动，本地启动的命令如下： 　

#如果设置好主题用此命令
hugo hugo server -D
#如果没有设置好主题用这个命令，xxxx是主题的名称
hugo server --theme=xxxx --buildDrafts

如果顺利执行会出现以下界面 在浏览器中输入图片红框中的地址进行预览效果，比如一般是localhost:1313。

建立GitHub仓库并将博客部署到GitHub上

建立GitHub仓库

  • 第一步

  • 第二步

  • 第三步,进入刚刚创建好的仓库，选择Settings,找到GitHubPages。

  • 第四步 ,Theme Chooser随便选择一个提交就可以了，之后会给你username.github.io这样的网址就表示实在对了，因为我们用public推送必须要这样的网址形式，不然会不显示CSS样式。

  • 第五步，去刚创建的仓库里的文件git到public文件夹中，里面有默认的配置文件，_config.yml、README.md必需要用到，下载下来之后复制到public文件夹中，之后就可以无障碍的push到你的项目中。

生成静态文件

接下来是生成静态文件，这里要把配置文件config.toml中的baseUrl修改为自己的仓库地址，即https：//XXXXXXXX.github.io/ 或者在hugo命令中使用指令指定 –baseUrl=“https：//XXXXXXXX.github.io/”

　 如果在配文件中已经设置好baseUrl和themes的路径，以及输出文件夹，可以直接在站点根目录下采用以下命令：

hugo

成功后会按照设置好的路径生成一个public文件夹，接下来我们需要将public文件夹中的内容提交到Github的仓库中去。

将本地文件部署到GitHub上去

#进入public文件夹
cd public
#初始化
git init
#添加public中的所有文件
git add .
#提交描述，XXXX中是你这次提交的描述，可以随便写
git commit -m "XXXXXXXX"
#链接到远端，把XXXX替换成自己的内容，不要忘了还有后缀.git
git remote add origin https://github.com/XXXXX/XXXXX.github.io.git

git pull origin master
git push -u origin master

整个过程应该不会有什么错误发生，实在要是解决不了可以使用下面这个命令强制上传

git push -u origin master -f

后续更新

后续再写文章的时候，直接在站点根目录下新建一篇文档就行了（如果像前边介绍的配置了生成模板的话会很方便使用的）：

hugo new post/xxx.md

如果对配置不做大的改动（例如：更换主题等），后续的push过程很简单的啦，只有下面几个了(根目录下使用命令)：

hugo
cd public
git add .
git commit -m "XXXXXXX"
git pull origin master
git push origin master

这里建议git命令使用git bash 上使用，不建议在cmd里面使用，因为总是会出现莫名其妙的bug
